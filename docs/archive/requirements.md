# 測量前チェックアプリ (CheckSheet) 機能・要件定義書

## 1. プロジェクト概要
本アプリケーションは、測量現場における事前準備や安全確認をデジタル化・効率化するためのPWA（Progressive Web App）です。紙ベースのチェックリストを排除し、現場の作業員がスマートフォンで直感的に操作できることを目的に設計されています。

## 2. ターゲットユーザーとユースケース
- **利用者**: 測量現場に向かう作業員・点検者
- **利用環境**: 電波が不安定、または完全に圏外になる可能性がある屋外（スマートフォン）
- **利用目的**: 測量前の必須確認事項（全43項目）を漏れなく確認し、完了報告用の出力を行う

## 3. 主要機能一覧

### 3.1 PWAおよびオフライン対応
- **完全オフライン動作**: Service Workerを利用した事前キャッシュにより、通信ができない山岳部やトンネル内でもすべての機能（回答、修正、PDF生成）が利用可能です。
- **ネイティブアプリライクな体験**: iOS/Androidに「ホーム画面に追加」することで、ブラウザのURLバー等を非表示にし全画面で動作します。
- **iOS向けインストール誘導 (InstallPrompt)**: PWAの強みを活かすため、iOS Safariでブラウザからアクセスしたユーザーに対し、自動的に「ホーム画面に追加する」案内を表示します。
- **自動アップデート**: ソースコードの更新時、次回オンライン環境でアプリを起動した際にバックグラウンドで最新版ファイルへ自動更新が行われます。

### 3.2 現場最適化UI/UX
- **直感的な操作（一問一答形式）**: 膨大なリストをスクロールするのではなく、1問ずつカード形式で表示し、認知負荷を下げます。
- **UIレイアウトの固定化**: 回答用のボタン（はい／いいえ）は常に画面下部に固定されています。回答のたびにボタンの位置がずれることがなく、親指を固定したままテンポ良く回答を進められます。
- **回答マトリックスとUndo（修正機能）**: 画面上部に回答状況を示す表（マトリックス）を常時表示。一度回答した質問であれば、いつでも任意の番号をタップして過去の問題に瞬時に戻り、再回答することが可能です。
- **全回答完了後の確認画面**: 最後の問題（43問目）に答えた後、即座に終了させず「全確認完了」のプレビュー画面を挟みます。ここでマトリックスから各問題を修正したのち、最終確定を行うことができます。
- **触覚フィードバック**: 「はい」「いいえ」のボタンをタップした際に僅かなバイブレーション（振動）を返し、画面を注視しなくても入力完了が伝わる設計です。
- **高品質SVGアイコン (Lucide-React)**: デバイス環境に依存してデザインがばらつく絵文字を廃止し、クリアで視認性の高いベクターアイコンを採用しています。

### 3.3 セッション管理と入力補助
- **オートセーブ（自動保存）**: 1問回答するごとに全体の進捗と状態を `localStorage` に自動保存します。誤ってアプリを閉じても、トップ画面の「続きから再開」機能で元の状態を復元できます。
- **ユーザープロフィールの永続化**: 各セッション開始時に入力する「現場名」と「点検者」は、初回の入力をデバイスに記憶させます。次回以降は自動で初期値としてセットされるため、入力の手間を省略できます。

### 3.4 レポート（PDF）出力機能
- **クライアントサイドでのPDF生成**: サーバーにデータを送信せず、端末の処理能力のみで完了報告用のPDFを生成します。
- **出力内容**: 現場名、点検者名、実施日時等の基本情報とともに、各カテゴリ別の「はい／いいえ」回答一覧が整形されて出力されます。

## 4. 技術スタック
- **基本フレームワーク**: React 19, Vite 5
- **PWA実装**: vite-plugin-pwa, Workbox
- **PDF生成**: html2canvas, jspdf
- **アイコンリソース**: lucide-react
- **デプロイ・ホスティング**: GitHub Pages (GitHub Actionsを用いた自動デプロイ)
