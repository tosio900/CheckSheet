# バリデーションメッセージの動的化と写真必須対応の実装計画

入力枠の名前が自由になり、写真添付が必須となるオプションが追加されたことに合わせ、ユーザーに対して適切なガイドを表示するように修正します。

## ユーザーレビューが必要な事項

> [!NOTE]
> メッセージの表示ロジックについて：
> 複数の項目が未入力の場合、すべてを列挙すると長くなる可能性があるため、「未入力の項目があります」といった形式にするか、最初の1つを表示する形式を検討しています。今回は「未入力の項目があります」または特定の項目名を表示する柔軟なロジックを提案します。

## 提案される変更点

### 1. チェック実行画面のロジック更新
バリデーションの状態をより詳細に取得し、コンポーネント間で共有します。

#### [MODIFY] [ChatCheck.jsx](file:///c:/Users/t-matsuki/Desktop/CheckSheet/src/components/ChatCheck.jsx)
- `isTextIncomplete` 判定時に、具体的にどのラベルが未入力かを抽出するロジックを追加。
- `AnswerControls` に `isTextIncomplete`, `isPhotoMissing`, `missingLabels` などのプロパティを渡すように変更。

### 2. 回答コントロールのUI更新
受け取ったバリデーション状態に応じて、メッセージを動的に生成します。

#### [MODIFY] [AnswerControls.jsx](file:///c:/Users/t-matsuki/Desktop/CheckSheet/src/components/check/AnswerControls.jsx)
- `isInputIncomplete` の代わりに詳細なフラグを受け取るように引数を更新。
- メッセージ生成ロジックの実装：
  - 写真が不足している場合：「写真を添付してください」
  - テキストが不足している場合：「『○○』を入力してください」
  - 両方不足している場合：「写真の添付と入力が必要です」
- スタイル調整（メッセージが長くなった場合の折り返しなど）。

## 影響範囲と注意点
- **UI/UX**: メッセージが動的になることで、ユーザーが「なぜ「はい」が押せないのか」を即座に理解できるようになります。
- **スタイリング**: モバイル端末での表示時にメッセージが収まるよう注意します。
