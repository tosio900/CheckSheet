# 現状の課題と今後の改善点 (Issues & Improvements)

本ドキュメントでは、測量前チェックアプリ（CheckSheet）の現状の技術的・UIUX上の問題点と、今後の開発フェーズで検討すべき改善項目を整理します。

## 1. 🚨 現状の問題点 (課題)

### 1.1 UIUXの限界
- **マトリックス表の長距離スクロール**: 上部に固定した回答一覧（マトリックス）は、全43問が横1列に並んでいるため、後ろの問題へアクセスする際の横スワイプ操作が負担になりやすい。
- **回答の「理由」を残せない**: 「いいえ」を選択した場合でも、なぜ「いいえ」なのか（具体的な不備の箇所など）をメモとして残す機能がない。

### 1.2 技術的な制約
- **LocalStorage依存データ保存のリスク**:
  - 現在、進捗データはブラウザの『LocalStorage』に保存されているが、ブラウザのキャッシュクリアや、iOS Safari自体の容量管理機能（長期間アクセスがないと自動削除される等）により、データが予期せず消えるリスクがある。
  - スマートフォン本体が故障・紛失した場合、復元ができない。
- **PDF出力の仕組み（画像化手法の限界）**:
  - `html2canvas` などの仕組みで「HTMLを画像（スクショ）化してPDFに貼る」という力技を行っているため、ファイル容量が数MBと不要に大きくなりやすい。
  - テキストが画像化されるため、PDF上の文字を選択したりコピーすることができない。
- **複数セッション管理の非対応**: 一度の中断で保持できるのは「1つの現場」のみであり、複数の現場を並行して行き来することができない。

---

## 2. 🚀 今後の改善・拡張計画 (Improvements)

### 2.1 高度なレポートとエビデンス機能
- **カメラ連携と写真添付**: 「いいえ」を選択した項目について任意でカメラを起動し、現場の写真を撮影（および簡単なメモを入力）できる機能を追加。PDF出力時にもエビデンスとして自動添付されるようにする。
- **ネイティブ準拠のPDF生成導入**: 画像に依存しない本格的なPDF生成ライブラリ（`pdfmake` などのフロントエンド完結型、またはヘッドレスブラウザを利用したバックエンド処理）への切り替えを行う。

### 2.2 オフライン・ファーストかつクラウド同期
- **DB自動同期機能 (Local-First Architecture)**:
  - IndexedDB等を用いてより強固にデバイス内にデータを永続化。
  - 電波がない場所では今まで通りオフラインで動作し、オンラインに復帰したタイミングでバックグラウンドで Firebase (Firestore) や Supabase 等のクラウドDBと自動同期させる。これにより、遠隔の事務所でも作業結果をリアルタイムに集計できるようになる。

### 2.3 UI/UXのさらなる最適化
- **マトリックスのブロック（タイル）化**: 横1列ではなく、縦・横に折り返して10区切りごとに表示されるタイル型UIに変更し、スクロール量と視認性を改善。
- **マルチ・プロジェクト機能**: 管理画面（ダッシュボード）を追加し、「未完の現場一覧」として複数のシートを並行管理できるようにする。

### 2.4 PWAの強化
- **Push通知機能**: PWAの強みを活かし、アプリのアップデート案内や、同期が未完了の現場がある場合にデバイスへプッシュ通知を送る機能を実装。
