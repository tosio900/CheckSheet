# UI/UX レビューレポート：CheckSheetプロダクト品質チェック

## 全体総評
モバイルでの現場利用を強く意識した設計となっており、ボタンのサイズ感や色の使い分けなど、基本的なアクセシビリティは良好です。
しかし、画面ごとの「揃え（整列）」のルールに一貫性がない点や、1px単位の微細なズレ、不要な視覚要素（スクロールバーなど）が残っており、正規リリースの品質としては「あと一歩」の磨き込みが必要です。これらを修正することで、現場の慌ただしい状況下でもより直感的に、かつ信頼感を持って使えるツールになります。

## 良い点
- **優れたタップターゲット**: チャット画面の「はい」「いいえ」ボタンが大きく設計されており、手袋をした状態や移動中でもミスなく操作できる。
- **直感的なカラー設計**: 緑（ポジティブ）と赤（ネガティブ）の使い分けに迷いがなく、結果画面での状況把握が非常にスムーズ。
- **機能的な情報集約**: 回答マトリックス（進捗管理）がつねに視界に入る設計は、ユーザーの心理的な負担を軽減している。

## 改善点

### 1. 入力情報画面の整列（視線の誘導）
- **対象箇所**: `StartScreen.jsx` (チェック開始前の情報入力画面)
- **問題**: タイトル周りが「中央揃え」なのに対し、入力欄のラベルやテキストが「左揃え」になっており、視線が不自然に左右に動かされる。
- **具体的な修正案**: すべての要素を左揃えに統一し、左端のインデントラインを一本に通す。見出しのみ中央にあるのは、特にモバイル画面では「情報の断絶」を感じさせる。
- **推奨する数値**: 
  - `.start-header`: `text-align: center` → `text-align: left`
  - 全体 Padding: `var(--space-6)` (24px) に基づく左端の整頓
- **優先度**: 高
- **期待効果**: 画面を開いた瞬間に「どこから入力すればいいか」が明確になり、認知的負荷が下がる。

### 2. 回答画面の不要なスクロールバー（視覚的ノイズ）
- **対象箇所**: `ChatCheck.jsx` (質問カード)
- **問題**: 質問文が短くスクロール不要な場合でも、カード右側に常にグレーのスクロールバーが表示されている、あるいは表示領域の隙間があるように見える。
- **具体的な修正案**: `overflow-y: auto` を適切に設定するか、カードの高さを内容量に合わせて可変にする（`min-height` のみ設定）。
- **推奨する数値**:
  - `.question-card`: `min-height: 140px`
  - `.check-content`: `overflow-y: auto` (不要な時は非表示にする設定の徹底)
- **優先度**: 中
- **期待効果**: アプリの完成度が上がり、不要な操作可能性（無駄なスワイプ）を排除できる。

### 3. プログレスバーの存在感向上
- **対象箇所**: `ChatCheck.jsx` 上部プログレスバー
- **問題**: 進捗を示す横棒が 8px と細く、屋外の太陽光下などでは現在の達成率がパッと見て判別しにくい。
- **具体的な修正案**: バーの厚みを増し、背景色とのコントラストを明確にする。また、バーの端を完全に丸めることでモダンな印象を与える。
- **推奨する数値**:
  - `.progress-bar-track`: `height: 12px`
  - `.progress-bar-fill`: 背景色をより鮮やかなグリーン（例: `#10B981`）に固定
- **優先度**: 高
- **期待効果**: 「あとどれくらいで終わるか」が直感的に伝わり、作業のモチベーション維持に寄与する。

### 4. 履歴一覧のテキストレイアウト（情報密度）
- **対象箇所**: `HistoryScreen.jsx` (履歴カード)
- **問題**: 「現場名」と「点検者名」が 1行に押し込まれており、長い場合に互いを圧迫している。また、アイコンと情報の間に余白が足りない。
- **具体的な修正案**: 現場名を 1行目、点検者名を 2行目に分けるか、現場名に `ellipsis`（三点リーダ）を適用してレイアウト崩れを防ぐ。
- **推奨する数値**:
  - 項目間 gap: `12px`
  - 現場名 max-width: `calc(100% - 40px)`
- **優先度**: 中
- **期待効果**: 過去のデータを探す際のスキャン性が向上し、誤読を防げる。

### 5. ホーム画面のボタン配置バランス
- **対象箇所**: `HomeScreen.jsx` (メインメニューボタン)
- **問題**: ボタン同士の間隔が 16px (`--space-4`) であり、全画面に対して要素が中央に固まりすぎて「窮屈」に見える。また、ボタン内のアイコンとテキストの垂直バランスがわずかにズレている。
- **具体的な修正案**: ボタン間の余白を広げ、垂直方向のリズムをゆったりとさせる。ボタンの高さも 4px ほど上げ、押しやすさとプレミアム感を両立させる。
- **推奨する数値**:
  - `.home-actions`: `gap: var(--space-6)` (24px)
  - `.btn`: `height: 60px` 程度を確保
- **優先度**: 低
- **期待効果**: アプリの「ファーストビュー」としての品格が向上し、プロ向けの堅牢なツールという印象が強まる。

## 最優先で直すべき5項目
1. **入力情報の左揃え統一**: 視線の蛇行を解消し、フォーム入力のストレスを無くす。
2. **プログレスバーの太型化 (12px)**: 現場での視認性を最優先で確保する。
3. **質問カードのスクロールバー制御**: 「未完成感」を即座に払拭する。
4. **ホームボタンの余白拡大 (24px)**: 誤タップ防止と安定感のあるレイアウトを実現。
5. **回答マトリックスの数値中央揃え**: 1px単位の配置ミスを修正し、精密機器のような信頼感を与える。
