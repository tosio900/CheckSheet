# 測量前チェックシステム 改善提案書（MVPレビュー後）

本ドキュメントは、MVPを実際に触ってみて得られたフィードバックと、実際の測量現場での過酷な作業環境（手袋の着用、片手操作、直射日光、雨天、時間的制約）を考慮し、システムをより実用的で現場に寄り添ったものにするための改善計画をまとめたものです。

---

## 1. ユーザーからのフィードバックに対する改善案

### 1.1 現場名・点検者の入力負担軽減
**【現状の課題】**
毎回手入力が必要であり、現場では文字入力自体が大きなストレスになる。手袋を外す手間も発生する。

**【改善アプローチ】**
- **過去の入力履歴の記憶と自動入力:** LocalStorageを用いて、最後に入力した「現場名」と「点検者」を記憶し、次回起動時に初期値として自動セットする。
- **履歴からの選択入力（ドロップダウン/サジェスト）:** 過去に入力された現場名・点検者のリストを保存しておき、タップだけで選択できるUIを提供する。
- **日付の完全自動化:** 初期値に今日の日付を入れるだけでなく、基本的に編集不要とし、システム側で自動で記録する（必要に応じて後から編集可能にする）。

### 1.2 備考の常時表示化
**【現状の課題】**
備考を確認するために「アコーディオンを開く」という1タップの操作が必要であり、手間がかかる。また、確認漏れが発生するリスクがある。

**【改善アプローチ】**
- **折りたたみの廃止:** 質問の下に初めから備考テキストを表示させる。
- **視覚的なメリハリ:** 質問文と備考文が混ざって読みづらくならないよう、備考は文字サイズを少し小さくし、背景色を変える（例：薄い黄色やグレーの枠）、または左にラインを引くなどして、「補足情報」であることが一目でわかるデザインにする。
- **重要キーワードのハイライト:** 備考内の「※許容誤差は２ミリ程度」「手入力するのは禁止」など、特に注意すべき箇所を太字や赤文字で強調する。

### 1.3 過去の回答履歴の表示（入力不安の解消）
**【現状の課題】**
一問一答形式は進めやすい反面、「今までの質問に何と答えたか」「間違って押してしまわなかったか」という不安が生じやすい。

**【改善アプローチ】**
- **タイムライン（チャットフィード）形式の導入:** LINE等のチャットアプリのように、過去の回答（直近3〜5問程度）が画面上部にスクロールして残るUIにする。
- **回答の視覚化:** 「Q1. ○○か？ → ✅はい」のようにコンパクトに履歴を表示し、間違えた場合はその履歴をタップすることで、直接その質問に戻って修正できるようにする。

## 2. 現場作業を極限まで考慮した追加改善提案

測量現場特有の環境要因（光、手袋、片手、通信環境など）を考慮し、以下の改善を提案します。

### 2.1 UI/UX（操作性・視認性）の向上
1. **ハプティック（振動）フィードバックの導入**
   - **理由:** 屋外で画面が反射して見えづらい場合、ボタンを押せたかどうかが視覚だけでは分かりにくい。
   - **対策:** 「はい」「いいえ」ボタンをタップした瞬間にスマートフォンを短く振動（バイブレーション）させ、身体感覚で入力を確認できるようにする。

2. **左右スワイプ操作の対応（Tinder風UI）**
   - **理由:** 厚手の手袋をしていると、どんなにボタンが大きくても押し間違えるリスクがある。
   - **対策:** 画面全体（または質問カード）を「右にスワイプで『はい』」「左にスワイプで『いいえ』」と操作できるようにする。これにより画面を見なくても片手でサクサク進められる。

3. **ハイコントラストモード / ダークモードの実装**
   - **理由:** 夏場の直射日光下では画面が白飛びして文字が読めなくなる。
   - **対策:** 背景を「真っ黒」、文字を「白や黄色」にしてコントラストを極限まで高めた「屋外（高輝度）モード」を用意する。

### 2.2 実務フローの効率化・的確化
1. **「いいえ」選択時の理由・写真・メモの強制/推奨**
   - **理由:** 「いいえ（＝異常あり、未実施）」が選択された場合、なぜ「いいえ」なのか証拠を残す必要がある。
   - **対策:** 特定の重要な質問で「いいえ」を選んだ場合、カメラが起動し現場の写真を撮るよう促すか、音声入力で理由を残せるようにする。（例: 丁張が壊れていた、図面が古かったなど）

2. **作業種別による動的な質問スキップ**
   - **理由:** 測量の種類（トラフ丁張なのか、起工測量なのか等）によっては、全く不要な質問セクションがあり、それを律儀に「はい/いいえ」と答えるのは時間の無駄。
   - **対策:** 開始時に「今回の主な作業」を選択させ、不要な質問（例えば後方交会法を使わないならそのセクション）を丸ごとスキップするロジックを入れる。

3. **NG（いいえ）項目の一括確認とアラート**
   - **理由:** 「いいえ」のまま作業を始めてしまうと事故や手戻りの原因になる。
   - **対策:** 最後の結果画面で、「いいえ」が一つでもあった場合、画面を赤く警告色にして「以下の問題が未解決ですが、本当に作業を開始しますか？」という強い注意喚起を行う。

### 2.3 データ管理・バックオフィス連携
1. **天気と気温の自動取得・記録**
   - **理由:** 測量機器（光波など）は気温や気圧の影響を受けるため、環境情報の記録が重要。
   - **対策:** スマートフォンの位置情報から現在地の天気・気温を自動取得し、チェックシートの記録として密かに持たせ、PDFに印字する。

2. **定期的なセルフチェック（キャリブレーション）のリマインダー**
   - **理由:** 測量機器自体の定期点検漏れを防ぎたい。
   - **対策:** アプリ側で「前回機器のキャリブレーションを行ってから○ヶ月経過しています」というアラートを出す機能（将来構想）。

## 3. 実装に向けたネクストアクション（ご提案プロセス）

今回のご要望（フェーズ1.5）を優先して実装するプランを立てます。

1. **入力負担の軽減（次回実装対象）**
   - `localStorage` に前回の現場名・点検者を保存し、初期表示に割り当てる処理を追加。
2. **備考の常時表示化（次回実装対象）**
   - `ChatCheck.jsx` のUIを変更し、アコーディオンを廃止。常時表示のカードデザインにスタイルを改修。
3. **過去履歴（直近5問）の表示（次回実装対象）**
   - 質問カードの上部に、回答済みリストを表示するエリアを新設。最新の回答が下に追加されていくチャットUIに変更。
4. **追加提案の中で即効性の高いものの導入（任意）**
   - 回答時の振動フィードバック（`navigator.vibrate` の追加）。

ご一読いただき、こちらの改善方針で実装を進めるか、他の案についても組み込みたいものがあればご指示ください。問題なければプログラムの修正に入ります。
