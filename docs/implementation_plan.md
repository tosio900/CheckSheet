# PDF改ページ時の文字切れ解消計画

PDF出力時のページ計算ロジックを修正し、画像やメモの量に関わらず文字が途切れないようにします。

## ⚠️ ユーザーレビューが必要
- 1ページあたりの項目数がこれまでより僅かに（1〜2個）少なくなる可能性があります。これは文字切れを防ぐための安全マージン確保によるものです。

## 提案される変更点

### 1. PDFレンダリングコンポーネントの修正

#### [MODIFY] [PDFTemplate.jsx](file:///c:/Users/t-matsuki/Desktop/CheckSheet/src/components/check/PDFTemplate.jsx)
高さ計算（ウェイト制）を以下の通り適正化します。

- **安全マージンの確保**: 
    - `PAGE_LIMIT` を `51` → `48` に引き下げ、ページ下部にバッファを持たせます。
- **ヘッダーウェイトの動的分算**:
    - メモの改行コード数に応じてウェイトを加算するように変更します。
- **画像ウェイトの計算改善**:
    - 画像1枚ごとの単純加算ではなく、横並び（最大3枚1行）を考慮した「行ベース」の計算に変更します。
    - 1行目のパディング分を考慮し、正確な高さをウェイトに反映させます。
- **項目の境界線**:
    - 画像がある項目とない項目で境界線の描画ロジックを整理し、視認性を維持しつつ計算誤差を減らします。

## 検証プラン

### 自動テスト / コード確認
- 重いデータ（長いメモ、最大数の画像添付）を入力した状態でPDFを生成し、ブラウザのデバッグモードで各ページの `.pdf-page`Div が `1244px` 以内に収まっているか目視・計測で確認します。

### 手動検証
- 以下のパターンでPDF出力を行い、ページ跨ぎの部分で文字・画像が切れないことを確認します。
    - メモが非常に長いケース
    - 画像が4枚以上（2行以上にわたる）あるケース
    - ページのちょうど境界に項目が来るケース
