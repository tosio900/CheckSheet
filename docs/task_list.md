# 残タスクリスト（検証・仕様決定フェーズ）

現在のコードベース上の実装（Phase 1 ～ Phase 5 および CI 導入）はすべて完了しています。今後は「実機での検証」ならびに「業務要件（仕様）の決定」がメインのタスクとなります。

## 1. 実機検証・環境依存タスク (UI/UX)
- [ ] **PNG画像の最適化（#2-10）**
  - 対象: `public/icon-512x512.png` などのアプリアイコン
  - 内容: Retina ディスプレイ等でぼやけない範囲で限界まで容量を圧縮（100KB以下への自動化など）。
- [ ] **iPad実機での PDF 出力テスト（#3-6）**
  - 対象: `PDFTemplate.jsx` および `pdfGenerator.js`
  - 内容: 実機の iPad Safari において、「複数ページにまたがる長い PDF（項目が多い場合）」を出力した際、下部が途切れることなくレンダリングされるか確認する。
- [ ] **PDF 生成アーキテクチャの根本修正（#3-7）**
  - 条件: 上記 `#3-6` の結果、`html2canvas` 方式で iPad 固有のメモリ制限・描画制限の限界が来た場合のみ着手する。
  - 内容: `html2canvas` への依存を捨て、`jsPDF-AutoTable` 等のネイティブ PDF 描画 API を使った直接生成方式へ書き換える。

## 2. 業務要件・仕様のフィードバック待ちタスク
現場で実際に使ってみたフィードバックを受けて判断する事項です。
- [ ] **複数ユーザーの共用端末運用（#S4）**
  - 内容: 一台の iPad を複数人で使い回す場合、「直近のチェックセッションがlocalStorageに残り続ける」のが不都合か確認。問題があれば明示的な「ログアウト・セッション破棄」機能を追加する。
- [ ] **過去の履歴閲覧対応（#S5）**
  - 内容: 「前回の現場のPDFをもう一度見たい」といった要望があるか確認。必要であれば、直近数件のセッション履歴をブラウザの IndexedDB に保存して一覧表示する History 画面を開発する。
- [ ] **個人情報のマスキング方針（#S2, #S3）**
  - 内容: PDFを出力する際、点検者名が個人情報として扱われる厳しい現場がないか確認。必要であれば氏名のイニシャル化・入力のマスキング機能を追加する。
